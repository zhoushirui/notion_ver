# notion_ver

纯前端小说 / 互动剧本写作工具。无后端、无数据库、无账号系统，数据以浏览器缓存作为草稿保护，以导出的 `.docx` / 备份 JSON 文件作为正式保存结果。

## 本地运行

```bash
npm install
npm run dev
```

## 部署到 Vercel

1. 把这个文件夹推到一个 GitHub 仓库。
2. 在 Vercel 里 "Import Project"，选中该仓库。
3. Vercel 会自动识别为 Vite 项目（本仓库也附带了 `vercel.json`），构建命令 `npm run build`，输出目录 `dist`，无需其他配置。
4. 部署完成后即可直接使用，无需环境变量。

也可以本地构建后把 `dist` 目录拖到 Vercel 的手动部署入口。

```bash
npm run build
```

## 使用说明

- **工作台页**：每个项目对应一本写作笔记，可新建 / 重命名 / 删除，支持批量导入本地文件夹为一本项目。
- **文档列表页**：管理该项目下的章节、大纲、灵感、废纸篓四类文档，支持类型修改、拖拽排序、单个或批量导出为 `.docx`，也支持从本地 `.docx` 导入。
- **编辑页**：左侧阅读或覆盖修改已有文档，右侧输入新内容，写完点击"保存"即追加到所选位置的文档末尾。输入框中输入 `/` 会弹出当前项目的角色姓名列表，回车或点击即可插入「【姓名】」并自动换行；列表最下方可添加新姓名，右键姓名可置顶或删除。工作台、文档列表和编辑页都会展示轻量字数统计。
- **数值面板**：每个项目可独立维护变量，例如角色好感、信任值、黑化值、存活值等，支持新增、删除、重命名、直接编辑分值和 `+1` / `-1` 微调。
- **常用片段**：每个项目可独立维护剧本模板片段，支持新增、编辑、删除，并可一键插入到右侧输入框当前光标位置。

## 数据安全说明

- 右侧输入框内容会在你打字时自动缓存到浏览器 `localStorage`，防止刷新或误关标签页丢失草稿；点击"保存"后草稿会被清空并合并进文档正式内容。
- 整个工作区（笔记本、文档、文档排序、角色列表、数值变量、常用片段）也会缓存在浏览器本地，方便下次打开时继续编辑，但**浏览器缓存不是可靠的长期存储**——请定期使用顶部的"导出备份"生成 JSON 备份包，或在文档列表页导出 `.docx`，作为正式保存结果。换电脑或清空浏览器数据前，务必先导出备份。
- "导入备份"会用备份文件中的内容整体替换当前工作区，导入前请确认；"批量导入"用于把本地 `.docx` / `.txt` / `.md` 文件夹导入为一本项目。
